# 线框图 · 电子校友卡（Alumni Card）

## 1. 页面目的与 US 映射

电子校友卡是校友端的"身份门面"，既是亮码通行凭证，也是校友荣誉表达载体。本页职责：

1. **动态 QR 亮码**：正面大尺寸 QR，每 30s 自动刷新 payload，支持离线缓存 5 分钟内的凭证以应对弱网。（US-008 / US-012）
2. **防截图水印**：全屏水印 + 跑马灯带 + 防截图 payload 绑定屏幕截图事件（小程序端监听 `onUserCaptureScreen` 告警）。（US-008）
3. **背面履历翻转**：点击卡面翻转展示荣誉徽章、就读履历、授权历史。（US-008）
4. **场景直达**：提供"入校闸机 / 活动签到 / 图书馆借阅 / 食堂折扣"等常用应用场景按钮，点击生成对应场景的专用短时令牌。（US-012）

覆盖用户故事：US-008 / US-012。

## 2. 设备与断点覆盖

| 设备 | 断点 | 布局 |
|------|------|------|
| 微信小程序 | 原生 | 全屏深色模式；亮度自动拉满；状态栏沉浸 |
| H5 移动端 | `mobile` | 与小程序同构；亮度 API 降级至提示"请手动调高屏幕亮度" |
| PC Web | `desktop` | 支持扫码登录或作为"我的证件盒"之一展示；卡面居中 420×260 |

本页禁用普通卡片 `shadow-md`，改用 `color-honor-gold` 描边 + 深色发光（详见 page-overrides.md）。

## 5. 交互关键点

1. **30s 倒计时刷新**：倒计时圈基于 `motion-qr-rotate` 动效，计时归零自动静默拉取 `GET /alumni/campus-card/qr`，失败则保留旧 QR 并显示 Badge "离线模式（剩余 X 分钟有效）"。
2. **防截图**：小程序端监听 `onUserCaptureScreen` 事件 → 立即作废当前 QR → 展示 Toast "检测到截图，二维码已失效" → 上报埋点。
3. **跑马灯水印**：CSS `animation` + 用户姓名 + 时间戳 + 随机漂移；`prefers-reduced-motion` 时改为静态对角线水印。
4. **亮度拉满**：进入本页调用微信 `wx.setScreenBrightness({ value: 1 })`；离开恢复原亮度。
5. **翻转动效**：3D CSS `rotateY(180deg)` 600ms；支持键盘 Enter 触发；`prefers-reduced-motion` 时改为左右滑入。
6. **场景按钮**：点击调用 `POST /alumni/campus-card/scene-token`，返回一次性短时令牌（15s 有效）替换主 QR 内容。
7. **空态**：首次进入若认证未通过，显示引导 "完成身份认证后即可使用电子校友卡" + CTA。
8. **错误态**：QR 获取失败连续 3 次时，引导 "请检查网络或截屏联系校友会客服"。
9. **加载态**：首次 QR 获取展示骨架圆 + "生成专属凭证中"；不超过 1.5s。
10. **深色模式**：本页强制深色（与 page-overrides.md 对齐），背景 `color-bg-primary` 深色值 `#0F172A`，卡面使用 `color-honor-gold` 描边 + 内发光。
