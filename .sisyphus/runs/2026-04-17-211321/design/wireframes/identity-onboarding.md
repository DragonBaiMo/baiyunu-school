# 线框图 · 校友身份认证漏斗（Identity Onboarding）

## 1. 页面目的与 US 映射

本页承接所有未认证访客进入平台后触发高权限入口时的身份核验，关键价值：

1. **五要素级联收集**：年级 → 学院 → 专业 → 班级 → 个人信息确认，五步强制漏斗，每步完成才能进入下一步。（US-006）
2. **挂起等待**：提交后进入 `reviewing` 挂起页，支持就地查询进度、微信消息通知、驳回重填闭环。（US-006）
3. **驳回申诉**：驳回态提供明确驳回原因、一键重填、"转人工申诉"绿色通道。（US-007 的前端呈现）
4. **认证通过**：成功页直接引导至电子校友卡激活。

覆盖用户故事：US-006 / US-007（前端呈现与状态回显，后端审核流在 AuditWorkflowPool PC 端）。

## 2. 设备与断点覆盖

| 设备 | 断点 | 布局 |
|------|------|------|
| 微信小程序 | 原生 | 单焦点全屏单步，顶部进度条永驻 |
| H5 移动端 | `mobile` | 同小程序 |
| PC Web | `desktop` 1024px+ | 居中单列 640px 宽度上限，背景分离式渐变，适合行政管理员自助注册 |

布局原则：**单页单焦点** —— 本页隐藏全局导航与 BottomBar，用户仅能专注当前步骤或退出。

## 3. 信息架构与区块优先级

| 优先级 | 区块 | 主职 |
|--------|------|------|
| P0 | 顶部进度条（5 步骤 + 当前步标签） | 让用户知道自己在哪里 |
| P0 | 当前步骤表单 | 单步信息采集 |
| P0 | 底部操作区（上一步 / 下一步 / 提交） | 步骤切换 |

## 5. 交互关键点

1. **级联联动**：年级变化会清空后续学院/专业/班级；学院变化清空专业/班级；专业变化清空班级。
2. **失焦校验**：手机号、身份证后 4 位失焦即校验格式；错误态 inline 红色文案 + `AlertCircle` 图标。
3. **保存草稿**：localStorage 持久化当前步骤与已填字段；重新打开自动恢复至最远一步，允许用户"回退修改"。
4. **空态**：首次进入无草稿时前 4 步下拉选项由 `GET /edu/grade-tree` 异步加载；加载态显示骨架下拉。
5. **加载态**：提交按钮 loading 期间禁用，并锁定上一步按钮避免重复提交。
6. **错误态**：网络错误显示 Toast "网络异常，请重试"，不清空已填数据。
7. **挂起轮询**：进入挂起页后每 30s 轮询 `GET /cert/:id`；后端状态变更推送 WS，收到即停止轮询。
8. **驳回回流**：点击"重新填写信息"回到表单第 1 步，后端返回的 `reason_field` 指示驳回具体字段并以 `color-danger` 标注。
9. **隐私同意**：未勾选协议时提交按钮禁用；勾选后记录时间戳 + 协议版本。
10. **全程无广告/无第三方跳转**：单焦点原则，隐藏 BottomBar / FAB / Banner。

## 7. 数据绑定映射

| 区块 | API 端点草案 | 方法 |
|------|-------------|------|
| 年级选项 | `/api/v1/public/edu/grades` | GET |
| 学院选项 | `/api/v1/public/edu/colleges?grade=` | GET |
| 专业选项 | `/api/v1/public/edu/majors?college=&grade=` | GET |
| 班级选项 | `/api/v1/public/edu/classes?major=&grade=` | GET |
| 提交认证 | `/api/v1/alumni/certifications` | POST |
| 查询工单 | `/api/v1/alumni/certifications/:id` | GET |
| 驳回重提 | `/api/v1/alumni/certifications/:id/resubmit` | POST |
| 人工申诉 | `/api/v1/alumni/certifications/:id/appeal` | POST |
| 上传辅证 | `/api/v1/alumni/uploads/cert-attachment` | POST（multipart） |
| WS 状态推送 | `wss://.../ws/cert/:id` | WS |
