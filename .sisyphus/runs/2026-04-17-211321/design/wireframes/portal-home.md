# 线框图 · 微门户首页（Portal Home）

## 1. 页面目的与 US 映射

微门户首页是校友触达平台的第一落点，承担四项职责：
1. **情绪建立**：首屏轮播传达母校资讯与荣誉成就，快速建立"亲切 + 荣誉感"心智（对应 US-002 展示场）。
2. **入口收敛**：金刚区承载高频业务入口（我的校友卡 / 入校预约 / 活动 / 捐赠 / 办事 / 更多），降低查找成本（对应 US-001 管理员 DIY 装配的前台呈现）。
3. **信息消费**：瀑布流新闻与活动列表支撑日常信息消费与检索（对应 US-002）。
4. **身份拦截**：对未认证用户在触发高权限入口时弹出拦截并导向身份认证漏斗。

覆盖用户故事：US-001 / US-002。

## 2. 设备与断点覆盖

| 设备 | 断点 | 核心布局 |
|------|------|---------|

| H5 移动端 | `mobile` 0–767px | 与小程序同构布局，底部 TabBar 承载"首页/办事/我" |

最小支持宽度 `mobile-xs` 320px（老年用户常见设备），首屏 CTA 不折行。

## 5. 交互关键点

1. **首屏 LCP 目标 < 1.8s**：Banner 首图预加载 + WebP 格式；金刚区 SVG Sprite 内联。
2. **搜索**：聚焦时下拉历史搜索 + 热门推荐；回车走 `/search?q=`；输入防抖 300ms。
3. **金刚区**：未认证用户点击"我的校友卡 / 入校预约 / 档案查询"直接触发拦截层，引导至身份认证漏斗。
4. **瀑布流**：IntersectionObserver 触发下一页；滚动到 80% 发起预取。
5. **空态**：新闻流为空展示默认 illustration + "校友会正在筹备首批资讯" + CTA "查看校庆专题"。
6. **错误态**：接口 5xx 时整块 fallback 为 `Toast` + 局部重试按钮，不整页崩溃。
7. **加载态**：骨架屏包裹 Banner / 金刚区 / 前 3 张卡片，持续时间不超过 1.2s 则直接跳过。
8. **FAB "亮码"**：对未认证用户点击弹窗"完成身份认证后可亮出专属电子校友卡"，CTA 跳转 `/identity/onboarding`；已认证用户直接推全屏亮码页。
9. **消息铃铛**：未读数徽章 `Badge variant="danger"`；超过 99 显示 `99+`。
10. **Banner 可访问性**：自动轮播在键盘聚焦内部时暂停；每张 slide 有 `aria-label` 描述。

## 7. 数据绑定映射

| 区块 | API 端点草案 | 方法 | 说明 |
|------|-------------|------|------|
| Banner 轮播 | `/api/v1/public/portal/banners` | GET | 匿名可访问；CDN 缓存 60s |
| 金刚区配置 | `/api/v1/public/portal/king-kong` | GET | 后台 DIY 装修器出参；含 `requireAuth` 标记 |
| 新闻瀑布流 | `/api/v1/public/portal/news-feed?cursor=&limit=10` | GET | 游标分页；默认 limit 10 |
| 搜索 | `/api/v1/public/search?q=&type=news\|activity` | GET | 走 Elasticsearch |
| 未读消息数 | `/api/v1/alumni/messages/unread-count` | GET | 需 JWT |
| 悬浮 FAB 亮码 | `/api/v1/alumni/campus-card` | GET | 身份拦截前置 |

错误模型引用 P2 架构蓝图 §5（RFC 7807 Problem+JSON）。

## 8. 埋点事件

| event_name | trigger | properties |
|------------|---------|-----------|
| `portal_home_view` | 首页首次渲染完成（LCP） | `{ lcp_ms, user_authed: boolean, entry_source }` |
| `portal_banner_click` | Banner 单张点击 | `{ slide_id, slide_index, title, link_url }` |
| `portal_kingkong_click` | 金刚区入口点击 | `{ entry_key, require_auth: boolean, authed: boolean }` |
| `portal_news_impression` | 新闻卡曝光 ≥ 50% 面积 | `{ news_id, position_index, exposed_ms }` |
| `portal_fab_lightcard_click` | 右下 FAB 点击 | `{ authed: boolean }` |
| `portal_search_submit` | 搜索提交 | `{ query, result_count, from: 'history'\|'manual' }` |

所有 `properties` 需通过埋点字典 schema 校验，未登记字段丢弃。
