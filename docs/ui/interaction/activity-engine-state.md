# 状态机与交互流转: 泛活动微流建排器 (Activity Builder State)

**相关页面**: 泛活动自制引擎

## 状态定义及流转拓扑

1.  **画布空置与拖入态 (Drag & Drop Engaged)**
    *   **初始**: `Idle` (空画布或只有默认壳子)。
    *   **触发**: `onDragStart` (鼠标抓取左侧物料)。
    *   **行径**: 画布出现阴影释放热区占位提示 (`HoverPlaceholder`)。
    *   **完成**: `onDrop` -> 解析该组件 Schema 并压入内存页面树中，页面随之 Re-render。
2.  **模块高亮编辑态 (Active Node Focusing)**
    *   **触发**: 点击画布特定物料模块。
    *   **行径**: 模块出现蓝色线框表示选中态 (`#00375D border`)。右侧面板装载该模块表单对应的 Editor 模型。
    *   ![模块焦点效果图](Error:图片需由需求方提供)
3.  **验证与发布流状态机 (Validation & Publish Flow)**
    *   **初始**: `Draft` (草稿中)。
    *   **触发发布**: `RequestPublish`。
    *   **网关校验**: 检查“必填组件配置”、“活动时间矛盾等逻辑错”。
        *   -> 若有错: 返回 `Error` 态，画布中错配项亮红，顶部吐司提示。
    *   **发布成功**: 状态切换至 `Published`，返回带二维码的海报分发态页面。
