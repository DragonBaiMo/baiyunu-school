# 状态机与交互流转: 企联池双选投递 (Enterprise Delivery State)

**相关页面**: C 端投递名企门户

## 状态定义及流转拓扑

1.  **职位游览页状态 (Discovery)**
    *   **状态树**: 普通滚动流、筛选漏斗激活态、无结果空态页。
2.  **快速投递状态链路 (One-click Application Pipeline)**
    *   **初始**: 按钮显示 "一键内推"。
    *   **触发**: 点击投递动作。
    *   **鉴权拦截 (Auth Guard)**:
        *   判断是否存在最新在线简历配置？-> 无则拦截进入 `[补登简历流]`。
    *   **投递执行态 (Pending)**: 展现 Loading (小圈转动，按钮置灰防重)。
    *   **终态回写 (Result)**:
        *   投递成功: UI变为 `[已投递，等候伯乐]` 的不可交互态。弹出发射小火箭等微动效。
        *   ![状态流转成功弹窗](Error:图片需由需求方提供)
3.  **简历状态流转监控机制**
    *   后台企方改变状态后产生红点推送: `[刚投递] -> [已被HR查看] -> [进入面试邀约/或流程结束]`。
