# 状态机与交互流转: 数据驾驶舱 (Dashboard State)

**相关页面**: 复杂数据分析驾驶舱

## 状态定义及流转拓扑

1.  **系统初始化加载极态 (Initializing)**
    *   **触发**: 用户进入大屏路由。
    *   **行径**: 加载 WebGL 场景与全局聚合接口。
    *   **流转**:
        *   -> `[加载成型 (Ready)]`: UI 完全就位，数字翻牌器开始第一次从 0 滚向真实值。
        *   -> `[降级加载 (Fallback)]`: 若性能过低或 3D 渲染失败，退化为 2.5D 切图地图或扁平图表。
2.  **数据极速流刷新 (Real-time Rehydration)**
    *   **触发**: 建立 WebSocket 信道，后端下发新的 `USER_ACTIVITY` 事件。
    *   **行径**: 触发 3D 沙盘落点动画，不刷新整个页面。
    *   **补充限制**: 每秒控制最多只消费 5 条高优动向，避免界面卡死。
3.  **模块下钻透视态 (Drill-down)**
    *   **触发**: 点击左/右侧图表中的特定切片 (例如某个省份的校友分布块)。
    *   **交互结果**: 呼出 Modal / 抽屉侧方弹出层，加载长明细列表 (分页数据)。
