## 1. 用户画像

| 角色 | 主要目标 | 使用频率 | 技术水平 |
|------|---------|---------|---------|
| **广大校友** | 获取母校资讯、联系同窗、办理返校预约与档案查询等事务 | 中 | 中 |
| **各级管理员**<br>*(一/二/分会)*| 审核校友资料、管理名录发布新闻、处理捐赠与后台配置 | 高 | 中 |
| **校友企业** | 在学校平台发布职位、招募校友人才 | 低 | 中 |

### 核心任务链
* **广大校友**：注册认证身份 → 浏览/检索母校资讯 → 报名校友活动/发起办事预约 → 参与在线现金/实物捐赠。
* **各级管理员**：工单审核（校友身份/入校预约） → 创建发布活动/新闻 → 查看宏观数据报表/监控捐赠荣誉榜。
* **校友企业**：完成实名企业资质认证 → 发布云招聘职位/宣讲会 → 接收并审阅校友简历。

## 2. 用户旅程

### 广大校友 旅程（Happy Path）

#### 场景：老校友听闻校庆活动，首次使用小程序进行身份认证、报名活动并预约返校

**旅程步骤：**
1. **发现平台** — 用户在微信收到校友会链接，打开“智慧校友服务”小程序首页 → 看到资讯轮播及校庆预告 → 尝试点击“校庆活动报名”。
2. **身份拦截与认证** — 系统弹出拦截，提示“请先完成校友身份认证” → 用户选择“表单自主认证”，提交学号、毕业年份等信息（US-001） → 系统提示“审核中”。
3. **获取电子校友卡** — 后台管理员（或对接统一身份认证）秒批通过 → 用户状态更新，在个人主页成功申领“电子校友卡”（带有动态核验二维码）（US-003）。
4. **报名活动与办事** — 用户返回活动大厅完成活动报名单填写（US-011） → 进入一站式办事大厅，提交含2位同行家属的返校参观预约（US-007）。
5. **返校核销签到** — 活动当日到达校门 → 亮出电子校友卡动态码 → 安保人员扫码核销放行（US-008） → 到达讲座现场，扫码完成活动签到。
6. **情感共鸣参与捐赠** — 参会期间受氛围感染，打开微端捐赠模块，小额直捐 200 元（US-013, US-014） → 获得电子捐赠证书且在大屏上快速滚动鸣谢（US-015）。

**情绪曲线：**
- 步骤 1-2: 期待但略显繁琐（需填写历史信息，担心记错学号）
- 步骤 3: 惊喜（审核极速通过，且获得了精美的专属电子名片）
- 步骤 4: 专注（顺利填报同行人和报备信息，流程清晰）
- 步骤 5: 顺畅与自豪（仅凭手机亮码无阻碍通行，科技感强）
- 步骤 6: 充满荣誉感与归属感（收获电子证书，看到名录闪过大屏）

**关键触点：**
- **最重要的交互节点**: 身份认证表单页（必须极简，支持容错与引导）；电子校友卡亮码页（必须支持高亮、防截图动态刷新）。
- **最容易失去用户的节点**: 身份信息填错并被打回重填的过程（如缺失历史对照库辅助）。

#### Edge Case（边缘场景）
- **认证持续受阻**：因曾用名或学号遗忘，无法通过自动查验，连续被转入手工驳回流，需有明显的“转人工申诉/客服申诉”绿色通道。
- **名额爆满/高并发限制**：大型校庆活动同时在线抢票，秒杀完后系统需锁死按钮，避免显示报满但仍可扣费或报错的情况；预约门禁当负荷超标需优雅地冻结日历灰显。

### 系统管理员 旅程（Happy Path）

#### 场景：行政管理员处理每日积压的校友申办工单，并发布新的捐赠项目

**旅程步骤：**
1. **总览宏观看板** — 登录 PC 端管理后台 → 首页展示“校友大数据可视化看板”（FR-003），快速了解今日新增注册及日活用户。
2. **批量审核认证工单** — 进入“认证审核工单池” → 核对上传证件及人脸底库（US-002） → 多选勾选，批量点击“通过”释放校友权限。
3. **活动/捐赠项目发版** — 进入捐赠管理模块，通过微流引擎图形表单新建“校友基金募款”（US-010, FR-010） → 上架前在手机端扫码预览效果 → 确认无误点击正式发布。
4. **大屏控场** — 校庆晚会现场 → 开启全屏可视化捐赠榜单走马灯（US-015） → 实时映出校友们的感恩弹幕。

**情绪曲线：**
- 步骤 1: 掌控感（平台报表清晰直观）
- 步骤 2: 疲劳但高效（人脸比对极大降低了审核难度，批量通过提高人效）
- 步骤 3: 严谨（预览机制保证不出差错）
- 步骤 4: 激动（看到实时效果和氛围推向高潮）

**关键触点：**
- **最重要的交互节点**: 工单池的批量勾选与审批对比页；拖拽式页面生成与预览器（US-005）。
- **最容易出错的节点**: 捐赠金额统计队列并发造成的漏播；不合规的页面推拉拽超出预设模块边框。

## 3. 页面清单 (Page Inventory)

### P0 级页面（首期核心与通路基石）

#### 【客户端 / 移动端】
*   **微门户首页** (列表页 / 导航页)
    *   **角色**: 广大校友
    *   **主任务**: 浏览母校资讯轮播与新闻列表（支持分类与搜索），寻找高频业务入口。
    *   **入口**: 微信小程序启动 / 扫码进入。
    *   **出口**: 点击新闻查阅详情，或点击顶部“我的校友卡”进入认证流。
*   **校友身份认证流** (表单页)
    *   **角色**: 待认证校友
    *   **主任务**: 填写学籍等必输信息或唤起统一身份登录授权，获取受限权限。
    *   **次任务**: 查看审核进度（审核中/已驳回重填）。
*   **电子校友卡与门禁码** (状态页 / 详情页)
    *   **角色**: 已认证校友
    *   **主任务**: 用于全屏动态亮码亮出身份，用作闸机出入、打卡及身份自证。
*   **活动大厅及详情表单** (列表及表单页)
    *   **角色**: 已认证校友
    *   **主任务**: 查询活动详情，拉起报名表单（带自定义题目如尺码忌口），形成电子电子票券并在现场核销出入校验。
*   **返校预约与办事大厅** (表单申请页)
    *   **角色**: 已认证校友
    *   **主任务**: 选择日历时间段，填写随行家属身份列表，申报进出校园行程。
*   **爱心捐赠大厅** (列表 / 收银表单及证书页)
    *   **角色**: 热烈回馈校友
    *   **主任务**: 挑选项目，发起支付流唤起三方收银台，完成后接收带编号电子捐赠感言证书大卡片。

#### 【管理端 / PC端】
*   **工单审批控制台** (列表 / 详情面)
    *   **角色**: 管理员（一级/二级）、安保人员
    *   **主任务**: 处理各类校友资料认证挂起池、返校入校白档库工单。带多选批通过、对比驳回答复功能。
*   **活动微流发布编辑器** (操作台)
    *   **角色**: 分会主理人、各类管理员
    *   **主任务**: 编辑发布表单问卷、设定限定群组资格参与控制。
*   **大屏数据可视化看板** (仪表板)
    *   **角色**: 一级行政统筹
    *   **主任务**: 管控校友基础分布画像及当前日活数据引擎流转清洗日志。

---

### P1 级页面（互动扩容与体验全维度升级）

*   **门户拖拽式装修后台** (PC端操作编辑器): 组合自定义首页不同节日版本，右侧调参数左侧响应。
*   **校友会群组话题交流区** (APP端/微端 帖子列表和详情区): 高级置顶、群删留言清理、发布同乡交流帖子。
*   **AI 智能政策咨询悬浮窗** (聊天挂件): 对接客服大脑知识库解决校友答疑并指派特定模块直达链接卡。
*   **在线电子学位补发与下载大厅** (表单/PDF水引呈现层): 完成行政盖章卷宗的数字资产领取提取保护界面。
*   **大型荣誉捐助光荣榜播报屏** (直播投影PC页): 全尺寸闪烁动效轮询隐名式脱敏实时流水账进账通报仪。
*   **校友企业与招聘发布主控** (列表页 / 企业控制台): （预留架构）企业宣讲流的挂载发布与职位库。

### P2 级页面（教育学习及附属增值池）

*   **大讲堂学习资源视频中心** (视频详情/目录): 对接“超星名师讲坛”视频列表框架等进阶功能。

## 4. 故事地图 (User Story Map)

### Backbone（用户主位工作流节点，横轴）

| 第一印象与身份绑定 | 资讯通览与社交互联 | 专属业务申请与发起 | 办事流转核销与反馈感 |
|--------------------|-------------------|-------------------|--------------------|

### Steps（每节点下归属拆分，纵轴）

**第一印象与身份绑定：**
- 获取自主认证申请权 (US-001)
- 后台人工/系统底库辅助审核 (US-002)
- 数字身份生成与电子校友卡亮出 (US-003)

**资讯通览与社交互联：**
- 各端自动适配排版浏览新闻 (US-004)
- 智能咨询助手获取策略 (US-006)
- 圈层群组的话题交流与信息治理 (US-012)
- 定制化大屏微门户组装 (US-005)

**专属业务申请与发起：**
- 行程回校同行人员造册表 (US-007)
- 多条件微表单线上报名活动生成 (US-010)
- 挑选项目并填入小额支付收银 (US-013, US-014)
- 在校经历数字证明寻回调提 (US-009)

**办事流转核销与反馈感：**
- 门禁及安保批量拦截/放行池审批 (US-008)
- 二维码检票与核查失效场控管理 (US-011)
- 虚拟数字大屏闪屏式爱心回馈鸣谢 (US-015)

### Release Slices（发布切片规划）

**首期 MVP 面向核心打通**
- **目标功能**: 全套基础认证链路，打通回校通道，基础捐方以及基本活动发布系统。
- **包含用户故事**: US-001, US-002, US-003, US-004, US-007, US-008, US-010, US-011, US-013, US-014。
- **页面范围**: 1. 微门户及认证表单；2. 电子校友卡核验展示码；3. 一站式大厅；4. 后台工单池和发版管理台；5. 在线支付模块组合。

**第二阶段 社区赋能与智能扩展**
- **目标功能**: 加入重运营功能，增强用户粘性和平台的智能化服务水平，丰富企业交互。
- **包含用户故事**: US-005, US-006, US-009, US-012, US-015。
- **页面范围**: 1. 拖拉式PC组合后台；2. 悬浮的 AI QA 框；3. 论坛式图文帖子大厅流及防屏蔽系统；4. 可拖拽电子防伪证明下载件；5. 投影电视款专属数字大屏端。

## 5. Storyboard (六格叙事场景)

**聚焦最高优先级流程：校友老李（毕业10年）预约返校并参与庆典**

*   **格 1: 场景设定** — 校庆前夕，老李计划带妻女回学校看看母校的剧变，但过去进校繁琐，不知如何获取许可权。
*   **格 2: 发现产品** — 他扫描同班群里丢出的“白云智慧校友平台”二维码，小程序闪出最新校庆资讯和“返校预约”核心大悬浮按钮。
*   **格 3: 核心操作** — 被拦截提示登录下，填入 2016 届毕业等数据秒批认证。顺畅在办事区填写了“星期六，随行家属2名”申请提交，随后又报了校长讲座活动。
*   **格 4: 系统反馈** — 当天下午老李的微信服务号叮咚提醒，“恭喜您返校审批通过，生成随行签到电子名片”。
*   **格 5: 价值实现** — 周末，老李全家在北门口仅出示动态数字校友卡，安保人员扫码台顺利响声“核验有效，欢迎回家”。讲座现场直接扫码签到。
*   **格 6: 下次再来** — 活动结束，老李颇受触动，在同程序下为教育基金顺手捐助，即刻截屏获得的专属水印电子纪念证书分享群聊，习惯性关注起了学校后续就业推介。
